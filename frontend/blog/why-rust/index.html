Why I build everything in Rust

April 1, 2025 • Rust

When people ask why I choose Rust for everything, they usually expect answers about performance or safety. Those are important, but they're not the whole story.

The real reason is simpler: Rust lets me focus on solving problems instead of fighting the language. The compiler catches my mistakes before users ever see them. The borrow checker prevents entire classes of bugs that plague other systems languages.

But here's what really matters: Rust gives me confidence. When I ship Rust code, I know it's not going to crash in production. I know it's not going to have memory leaks. I know it's not going to have data races.

The Learning Curve

Yes, Rust has a learning curve. The borrow checker is challenging at first. But once you understand it, you start thinking differently about memory management. You become a better programmer, not just in Rust, but in any language.

Compare that to other languages where you're constantly fighting runtime errors, null pointer exceptions, and memory leaks. Rust pushes the complexity to compile time, where it belongs.

Real-World Benefits

In production, Rust code just works. It's fast, it's memory-safe, and it's reliable. I've built web servers, CLI tools, and even desktop applications in Rust. They all share one thing in common: they don't crash.

That's not a luxury. That's a requirement for production software. Users don't care about your elegant architecture or your clever algorithms. They care that your application works when they need it.

Conclusion

I build everything in Rust because it lets me build better software. Software that's fast, safe, and reliable. Software that I can ship with confidence.

That's not a replacement story. That's a collaboration story—and it's far more exciting.
